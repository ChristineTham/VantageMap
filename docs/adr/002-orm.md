# ADR-002: ORM and Migration Tooling

**Status:** Accepted  
**Date:** 2026-05-12  
**Decision:** Drizzle ORM  

## Context

VantageMap needs a TypeScript ORM/query builder for PostgreSQL (ADR-001) that supports:
- Type-safe queries with TypeScript strict mode
- Database migrations with version control
- JSONB column support for custom fields
- Raw SQL escape hatch for complex queries (recursive CTEs, full-text search)
- Serverless/edge compatibility (Neon serverless driver)
- Next.js 16 App Router compatibility (no global connection pool issues)
- AI agent familiarity for vibe coding

## Options Evaluated

### 1. Drizzle ORM

| Criterion | Assessment |
|-----------|------------|
| License | Apache 2.0 |
| Type safety | Schema-first with full TypeScript inference; SQL-like API |
| Migrations | `drizzle-kit` CLI with `push`, `generate`, `migrate` commands |
| JSONB support | Native `jsonb()` column type with typed inference |
| Raw SQL | `sql` template literal for arbitrary SQL |
| Serverless | Zero dependencies, works with Neon serverless driver |
| Bundle size | ~33 KB (minimal, no runtime overhead) |
| Relational queries | `db.query.*` API for nested/relational data fetching |
| Auth integration | Auth.js adapter, Better Auth adapter (first-class) |
| AI familiarity | Very high — dominant in modern Next.js tutorials since 2024 |

### 2. Prisma

| Criterion | Assessment |
|-----------|------------|
| License | Apache 2.0 |
| Type safety | Schema-first with generated client; excellent autocomplete |
| Migrations | `prisma migrate` with diff-based migration generation |
| JSONB support | `Json` type (less typed than Drizzle; runtime validation needed) |
| Raw SQL | `$queryRaw` and `$executeRaw` template literals |
| Serverless | Requires Prisma Accelerate or Data Proxy for connection pooling |
| Bundle size | Heavy — Rust-based query engine (~15 MB binary) |
| Query patterns | Abstracted from SQL; own query language |
| Auth integration | Auth.js adapter (first-class), Better Auth adapter |
| AI familiarity | Very high — historically most popular TypeScript ORM |

**Concerns:**
- **Prisma Client generation step** adds build complexity for CI and vibe coding
- **Query engine binary** (~15 MB) bloats serverless functions and increases cold start
- **Connection pooling** requires Prisma Accelerate (paid) or external PgBouncer for serverless
- **JSONB typing** requires manual Zod validation; no compile-time type safety on JSON contents
- **SQL distance** — Prisma's query API is far from SQL, which makes debugging and optimization harder

### 3. Kysely

| Criterion | Assessment |
|-----------|------------|
| License | MIT |
| Type safety | Interface-first type definitions; SQL-like query builder |
| Migrations | Basic file-based migrations (manual up/down) |
| JSONB support | `JSONColumnType` wrapper for typed JSON columns |
| Raw SQL | `sql` template literal |
| Serverless | Lightweight, works with pg driver |
| Bundle size | ~50 KB |
| Auth integration | Auth.js adapter |
| AI familiarity | Moderate — less tutorial coverage than Drizzle or Prisma |

**Concerns:**
- **No schema push/pull** — migrations must be written manually
- **No relational query API** — joins must be written explicitly (fine for SQL experts, harder for AI agents)
- **Smaller ecosystem** — fewer plugins, adapters, and community resources
- **Type definitions are manual** — no code generation from database, must maintain types by hand

### 4. TypeORM

| Criterion | Assessment |
|-----------|------------|
| License | MIT |
| Type safety | Decorator-based entities (relies on `reflect-metadata`) |
| Migrations | Auto-generate and manual migrations |
| JSONB support | `@Column({ type: 'jsonb' })` |
| Raw SQL | `query()` method |
| Serverless | Not designed for serverless; connection pool management issues |

**Disqualified:** TypeORM uses decorators and `reflect-metadata` which conflict with Next.js App Router and React Server Components. Its connection management is not designed for serverless. The project has had periods of slow maintenance. Not recommended for new TypeScript projects.

## Decision

**Drizzle ORM** is selected as the ORM and migration tooling.

### Key factors:

1. **SQL-like API** — Drizzle's query builder reads like SQL, making it easy for AI agents to generate correct queries and for developers to debug them
2. **Zero dependencies** — no binary engine, no generation step; works immediately in serverless and edge environments
3. **First-class serverless** — works natively with Neon's serverless driver, no connection pooling proxy needed
4. **Schema-as-code** — TypeScript schema files are the source of truth; `drizzle-kit` generates SQL migrations automatically
5. **JSONB with types** — `jsonb().$type<CustomFieldSchema>()` provides compile-time type safety on JSON contents
6. **Better Auth integration** — Better Auth (ADR-003) has a first-class Drizzle adapter with schema generation
7. **Active ecosystem** — 33K+ GitHub stars, actively maintained, strong community adoption in Next.js projects
8. **Vibe-coding optimized** — no `prisma generate` step means one less thing for AI agents to forget; schema changes take effect immediately

### Drizzle schema example for VantageMap:

```typescript
// src/db/schema/capabilities.ts
import { pgTable, text, uuid, integer, jsonb, timestamp } from 'drizzle-orm/pg-core';

export const businessCapabilities = pgTable('business_capabilities', {
  id: uuid('id').primaryKey().defaultRandom(),
  name: text('name').notNull(),
  description: text('description'),
  level: integer('level').notNull().default(1),
  parentId: uuid('parent_id').references(() => businessCapabilities.id),
  lifecyclePhase: text('lifecycle_phase').notNull().default('active'),
  healthStatus: text('health_status'),
  customFields: jsonb('custom_fields').$type<Record<string, unknown>>(),
  createdAt: timestamp('created_at').defaultNow().notNull(),
  updatedAt: timestamp('updated_at').defaultNow().notNull(),
});
```

### Migration workflow:
```bash
npx drizzle-kit generate  # Generate SQL migration from schema changes
npx drizzle-kit migrate   # Apply pending migrations
npx drizzle-kit push      # Push schema directly to dev database (no migration file)
npx drizzle-kit studio    # Visual database browser (free)
```

## Consequences

- Phase 3 schema files live in `src/db/schema/` as TypeScript files
- `drizzle-kit` manages migrations in `drizzle/` directory
- Connection configuration uses Neon serverless driver for MVP, `pg` pool for production
- Better Auth adapter generates auth-related schema automatically
- AI agents can write Drizzle queries that closely resemble SQL, reducing hallucination risk
